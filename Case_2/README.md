# AI4Bat 案例 2：材料研发的三大范式 —— 以电池抗枝晶材料设计为例

在电池研发中，如何快速找到性能优异的新材料是核心挑战。本案例将以**剪切模量 (Shear Modulus)** 的预测为切入点，带领大家体验材料科学研究的三种主要范式：**实验试错**、**计算指引 (DFT)** 以及 **ML 辅助 (机器学习)**。

---

## 背景：为什么要关注剪切模量？

在锂金属电池中，锂枝晶 (Lithium Dendrite) 的生长会刺穿隔膜，导致电池短路甚至起火。根据 **Monroe-Newman 准则**，如果固态电解质或涂层的剪切模量 $G$ 大于金属锂剪切模量的两倍（锂的 $G \approx 4.2 \text{ GPa}$），就能在力学上有效抑制枝晶的生长。

因此，寻找高剪切模量的材料是提升电池安全性的关键。

---

## 材料研发的三大范式对比

| 范式 | 核心手段 | 周期 | 成本 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **实验试错** | 合成 + 表征 | 数月/年 | 高 | 结果最真实 | 盲目性强、效率低 |
| **计算指引** | 物理建模 (DFT) | 数天/周 | 中 | 精度高、可预测未知 | 资源消耗大、模型简化 |
| **ML 辅助** | 数据驱动 (AI) | 秒/分 | 低 | 速度极快、全空间搜索 | 依赖数据质量、可解释性弱 |

## 实践环节

### 1. 计算指引 (DFT 实战)
我们以 **LiF (氟化锂)** 为例，通过第一性原理计算 (DFT) 预测其力学性质。

- **关键文件**：
  - [POSCAR](DFT/POSCAR): 描述 LiF 的晶体结构。
  - [INCAR](DFT/INCAR): 设置 DFT 计算参数（如交换关联泛函、能量截断等）。
  - [OUTCAR](DFT/OUTCAR): 计算输出结果文件。
- **计算结果分析** (由 [mechanical_properties.dat](DFT/mechanical_properties.dat) 导出)：
  - **体积模量 (Bulk Modulus, B)**: **64.2 GPa**
  - **剪切模量 (Shear Modulus, G)**: **46.5 GPa** (Hill 平均)
  - **结论**：LiF 的剪切模量远高于抑制锂枝晶所需的阈值（~8.4 GPa），在力学上是理想的界面保护层。

### 2. ML 辅助 (数据驱动预测)
我们将利用 `matminer` 库和已有的大型材料数据库（如 Materials Project），构建一个预测剪切模量的机器学习模型。

- **数据来源**：Matminer 提供的弹性模量数据集 (`elastic_tensor_2015`)。
- **核心流程**：
  1. **特征提取**：利用 `ElementProperty` 等工具将化学组成转化为特征向量。
  2. **模型构建**：训练随机森林 (Random Forest) 或梯度提升树模型。
  3. **评估验证**：通过 $R^2$ 得分和均方根误差 (RMSE) 评估模型精度。
- **实验脚本**：
  - [01_ml_shear_modulus.py](01_ml_shear_modulus.py): ML 完整预测脚本。
  - [ML_README.md](ML_README.md): ML 部分详细操作指南。

## 思考与讨论

1. **训练好的 ML 模型如何高效使用？**
   - 提示：模型训练完成后，可以对包含数百万种材料的数据库（如 Materials Project 或未知化学空间）进行“秒级”预测，从而实现高通量筛选。
2. **如果传统 ML 模型预测误差太大怎么办？**
   - 提示：可以考虑增加特征工程的复杂度（如引入结构描述符）、使用深度学习模型（如图神经网络 GNN）、或者采用“主动学习 (Active Learning)”通过少量高精度 DFT 计算来迭代修正模型。
