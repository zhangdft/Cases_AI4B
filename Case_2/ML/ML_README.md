在这一阶段，我们不仅构建一个预测模型，还将深入探讨数据驱动研发的核心步骤：特征工程、多模型对比以及特征去冗余。

## 实验内容：剪切模量预测实战 (高级)

**对应脚本：`01_ml_shear_modulus.py`**

### 1. 特征查看与探索 (Feature Exploration)
在建模之前，了解数据是至关重要的。
- **目标分布**：查看剪切模量 $G$ 的数值分布（生成 `target_distribution.png`）。
- **相关性分析**：找出与剪切模量相关性最强的物理化学描述符（如电负性、熔点等）。

### 2. 特征去冗余 (De-redundancy)
原始提取的特征（Magpie 预设）包含 130 多个描述符，其中很多特征之间存在极高的线性相关性（冗余）。
- **方法**：计算特征间的皮尔逊相关系数。
- **操作**：自动删除相关性大于 0.95 的特征对中的一个。
- **意义**：减少模型过拟合风险，提升计算效率，使模型更具解释性。

### 3. 多模型对比 (Model Comparison)
没有一种算法是万能的。本脚本对比了以下四种常见模型：
1. **Linear Regression (线性回归)**：基础模型，考察特征与目标是否呈线性关系。
2. **Random Forest (随机森林)**：集成学习的代表，处理非线性关系能力强。
3. **XGBoost (极端梯度提升)**：目前竞赛中最常用的高效算法，通常具有极高的精度。
4. **SVM (支持向量机)**：利用核函数处理高维非线性问题（需进行数据标准化）。

**可视化结果**：生成 `model_comparison.png`，通过箱线图直观展示各模型的 $R^2$ 分布情况。

### 4. 最佳模型训练与验证
脚本会自动选定 $R^2$ 得分最高的模型进行最终训练，并生成预测值 vs 真实值的 **等值线图 (Parity Plot)** (`final_model_parity_plot.png`)。

## 教学目标

- 理解 **特征工程** 中去冗余的重要性。
- 掌握如何使用 **交叉验证** 公平地对比不同 ML 算法。
- 学习针对不同算法（如 SVM）进行 **数据标准化 (Standardization)** 的必要性。
- 培养从“黑盒模型”转向“科学分析”的思维习惯。
